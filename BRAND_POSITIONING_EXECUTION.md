## 第六部分：关键信息点

### 6.1 必须反复传达的 3 个核心点

**Point 1: Not a Framework**
```
"UniAdapter 不是框架。
我们是一个适配层。
给你现有项目的多端能力。"
```

**Point 2: Seamless Integration**
```
"5 分钟集成。
不改变你的代码结构。
不学习新 API。
完全保留现有投入。"
```

**Point 3: For Existing Projects**
```
"已经有 React 项目了？
已经有 Vue 项目了？
想要多端支持？
UniAdapter 就是为你准备的。"
```

### 6.2 必须回答的常见问题

**Q: UniAdapter 和 uniapp 有什么关系？**
```
❌ A: "我们是改进的 uniapp"
✅ A: "没有关系。我们是完全不同的东西。
        uniapp 是框架，UniAdapter 是工具。
        uniapp 用于新项目，UniAdapter 用于现有项目。"
```

**Q: 为什么不是框架？**
```
✅ A: "框架强制你学习新 API 和配置。
        我们只是提供适配，让你继续用自己熟悉的框架。
        这样学习成本最小，集成最快。"
```

**Q: 与 Taro 的区别？**
```
✅ A: "Taro 是一个框架。UniAdapter 是一个工具。
        如果你已有现成项目，Taro 无法直接集成，
        但 UniAdapter 可以。"
```
